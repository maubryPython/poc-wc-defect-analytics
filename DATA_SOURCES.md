# Data Sources

All data used in this POC is **synthetic** (generated by script) or **open-source**.
No real Nike data, proprietary supplier data, or personally identifiable
information is used anywhere in this project.

Generation script: `data/generate_synthetic_data.py` (seed=42, fully reproducible)

---

## Datasets

### 1. `supplier_manifest.csv`
| Field | Type | Description | Source |
|---|---|---|---|
| supplier_id | string | Unique supplier code (S001–S008) | SYNTHETIC |
| supplier_name | string | Fictional company name | SYNTHETIC |
| region | string | Vietnam / Indonesia / Bangladesh | SYNTHETIC |
| material_type | string | jersey_fabric / bonded_seam_tape / heat_transfer_film | SYNTHETIC |
| contract_tier | string | tier1 / tier2 | SYNTHETIC |
| baseline_defect_rate | float | Expected defect rate under normal conditions | SYNTHETIC |

**Notes:** S007 (Surabaya Bonding Tech) and S008 (Ho Chi Minh Film Coat) are the
suppliers involved in the defect incident. Their baseline rates are intentionally
elevated to provide signal for the Detection Agent's supplier-context lookup.

### 2. `supplier_defect_history.csv`
| Field | Type | Description | Source |
|---|---|---|---|
| supplier_id | string | Links to supplier_manifest | SYNTHETIC |
| line_id | string | Production line (L1–L4) | SYNTHETIC |
| month | string | YYYY-MM format | SYNTHETIC |
| units_inspected | integer | Units checked that month | SYNTHETIC |
| defect_rate | float | Observed rate for that month | SYNTHETIC |
| defects_found | integer | Computed: units_inspected × defect_rate | SYNTHETIC |

**Notes:** Covers Jan 2025 – Mar 2026 (15 months). S007 and S008 show a
statistically detectable creeping trend on lines L2/L3 beginning Dec 2025.
This is the baseline the Detection Agent's `baseline_lookup` tool queries.

### 3. `manufacturing_batches.csv`
| Field | Type | Description | Source |
|---|---|---|---|
| batch_id | string | Unique batch code (B-001 to B-200) | SYNTHETIC |
| supplier_id | string | Links to supplier_manifest | SYNTHETIC |
| production_date | date | ISO format, Jan 5 – Mar 28 2026 | SYNTHETIC |
| line_id | string | L1–L4 | SYNTHETIC |
| unit_count | integer | Units in this batch (500–2000) | SYNTHETIC |
| material_lot | string | Lot tracing code | SYNTHETIC |
| material_type | string | Inherited from supplier | SYNTHETIC |
| sku | string | WC-HOME-M / WC-AWAY-M / WC-HOME-W / WC-AWAY-W | SYNTHETIC |
| **is_defective** | boolean | **GROUND TRUTH — DO NOT ingest into pipeline** | SYNTHETIC |

**⚠️ Important:** The `is_defective` column is ground truth for evaluation only.
Strip this column before uploading to the Databricks landing zone. The agents
must discover defective batches through QC data alone.

14 of 200 batches are defective: S007/S008 batches on L2/L3, produced Jan 15 – Feb 28 2026.

### 5. `sales_returns.csv`
| Field | Type | Description | Source |
|---|---|---|---|
| transaction_id | string | TXN-000001 format | SYNTHETIC |
| date | date | Feb 1 – Apr 20 2026 | SYNTHETIC |
| sku | string | WC-HOME-M / WC-AWAY-M / WC-HOME-W / WC-AWAY-W | SYNTHETIC |

| units_returned | integer | Derived from return_rate | SYNTHETIC |
| return_rate | float | 2–6% pre-discovery; 18–35% post-April 8 | SYNTHETIC |
| return_reason | string | wrong_size / changed_mind / product_defect / item_damaged | SYNTHETIC |
| revenue_usd | float | units_sold × ~$99 avg price | SYNTHETIC |
| refund_usd | float | units_returned × ~$99 avg price | SYNTHETIC |

**Notes:** Return rates spike sharply on April 8 2026 (defect discovery date),
providing the Scenario Agent with a before/after signal for financial modeling.

## Reproducibility

All datasets are generated with `numpy.random.seed(42)` and `random.seed(42)`.
Re-running `data/generate_synthetic_data.py` produces bit-identical output.
